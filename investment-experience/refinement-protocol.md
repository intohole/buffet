# 经验锤炼协议

经验不是记录就结束，而是需要持续锤炼、验证、升级，最终融入决策规则。

## 锤炼闭环

```
记录经验 → 交叉验证 → 置信度提升 → 规则化 → 融入决策框架
    ↑                                                    |
    └────────── 反证降级 ←── 矛盾发现 ←── 持续检验 ←─────┘
```

## 置信度升级规则

| 当前置信度 | 升级条件 | 降级条件 |
|-----------|---------|---------|
| 低(1次验证) | 再获1次独立验证→中 | 1次反证→标注"待验证" |
| 中(2-3次验证) | 再获2次独立验证→高 | 1次反证→降为低 |
| 高(4次+验证) | 可写入决策规则 | 2次反证→降为中 |

**验证来源权重**：

| 来源类型 | 权重 | 说明 |
|---------|------|------|
| 实战结果(盈亏验证) | 3x | 最可靠的验证 |
| 多源网络搜索交叉验证 | 2x | 次可靠 |
| 大师方法论支持 | 1.5x | 理论支撑 |
| 单一来源 | 1x | 基础验证 |

## 认知升级引擎

### 规则提炼路径

```
原始观察 → 模式识别 → 假设形成 → 验证测试 → 规则提炼 → 决策框架融入

示例:
原始观察: "涨停潮后次日追涨必亏"
→ 模式识别: "板块涨停>10股后次日分化概率>90%"
→ 假设形成: "涨停潮=短期过热信号"
→ 验证测试: 回溯3次涨停潮后走势
→ 规则提炼: "涨停潮后不追高，等回调至MA20"
→ 决策框架融入: 写入SKILL.md热点选股漏斗Phase 2
```

### 认知偏差修正

| 偏差类型 | 表现 | 修正方法 |
|---------|------|---------|
| 确认偏差 | 只关注支持自己判断的证据 | 强制搜索反面观点 |
| 锚定效应 | 过度依赖首次获取的信息 | 每次重新评估所有信息 |
| 后视偏差 | "我早就知道会这样" | 记录判断时的置信度和理由 |
| 过度自信 | 高估自己判断的准确性 | 追踪判断准确率，用数据说话 |
| 损失厌恶 | 亏损时不愿止损 | 严格执行止损纪律，不例外 |

## 反脆弱机制

从错误中学习，让每次失败都成为系统进化的养分。

### 错误分类与应对

| 错误类型 | 定义 | 应对策略 |
|---------|------|---------|
| 执行偏差 | 知道正确做法但没做到 | 强化纪律检查清单 |
| 认知偏差 | 分析框架有缺陷 | 修正框架/阈值 |
| 信息不足 | 关键数据缺失导致误判 | 增加搜索维度 |
| 系统性风险 | 不可控的外部冲击 | 降低仓位，增加安全边际 |
| 运气成分 | 正确的判断但结果不好 | 不修正规则，记录为统计噪声 |

### 错误复盘协议

每次判断错误后，必须执行：

1. **归因分析**: 属于上述5种错误中的哪一种？
2. **根因追溯**: 为什么会产生这个错误？(5 Why分析法)
3. **修正动作**: 具体要改变什么？(规则/阈值/流程)
4. **验证计划**: 如何验证修正是否有效？
5. **经验沉淀**: 将修正后的规则写入对应经验文件

### 50笔决策质量追踪

- 回溯最近50笔分析，每笔按-100至100分打分
- 得分<-30的交易，强制写出3点修正动作
- 将得分绘制成折线图，观察趋势方向
- 趋势上行=系统在进化，趋势下行=系统需要检修

## 经验锤炼触发条件

| 触发条件 | 锤炼动作 |
|---------|---------|
| 每次分析后 | 检查是否有新经验可沉淀 |
| 判断被证实时 | 提升经验置信度 |
| 判断被证伪时 | 降低置信度+归因分析+修正 |
| 每月策略复盘 | 统计经验命中率，调整规则权重 |
| 发现新规律时 | 新增经验条目，标注低置信度 |
| 高置信度经验≥5条时 | 考虑写入决策规则(SKILL.md) |
