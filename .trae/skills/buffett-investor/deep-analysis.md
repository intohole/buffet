# Deep Analysis - 深度分析与选股模块

包含3种分析模式: 选股筛选、买卖时机、持仓诊断。

## Mode A: 选股筛选 (Stock Screening)

### Step 1: 确定筛选条件

| 条件 | 选项 | 默认 |
|------|------|------|
| 风险偏好 | 保守/均衡/进取 | 均衡 |
| 投资期限 | 短期(<1月)/中期(1-3月)/长期(>1年) | 中期 |
| 板块偏好 | 指定板块/全市场 | 全市场 |
| 资金规模 | 金额 | 10万 |

### Step 2: 多源数据采集

执行4批次并行WebSearch：

**批次1 - 价值筛选**:
```
- "A股 低PE 高ROE 高股息 价值股 {当前年份}"
- "护城河 宽护城河 龙头股 {当前年份}"
- "巴菲特式 A股 价值投资 标的 {当前年份}"
```

**批次2 - 成长筛选**:
```
- "A股 高增长 PEG<1 成长股 {当前年份}"
- "净利润增速>30% 一季报 {当前年份}"
- "机构调研 最密集 {当前月份}"
```

**批次3 - 股息筛选**:
```
- "A股 高股息 股息率>5% {当前年份}"
- "红利指数 成分股 分红稳定"
```

**批次4 - 板块筛选**:
```
- "[指定板块] 龙头股 核心标的 {当前年份}"
- "[指定板块] 业绩超预期 券商推荐 {当前年份}"
```

### Step 3: 5层过滤

**Layer 1 风险与纪律** (前置必做):
- 能力圈检查: 能否用3句话解释该业务？
- 情绪检查: 是否出于恐惧/贪婪/FOMO？
- 未通过→淘汰

**Layer 2 门控** (最关键，淘汰率最高):
- 财务质量≥5/7项合格
- 护城河≥5分
- **(反馈闭环)** 若淘汰率>80%→放宽标准重新筛选

**Layer 3 分类**:
- 正确识别林奇6型
- PEG估值合理

**Layer 4 时机**:
- 买入信号≥4/6

**Layer 5 宏观**:
- 环境有利

### Step 4: 排名与选择

**综合评分公式**:

| 因素 | 权重 | 评分方法 |
|------|------|----------|
| 护城河 | 30% | 0-15分制 |
| 财务质量 | 25% | 0-7项合格 |
| 安全边际 | 20% | 内在价值-当前价格 |
| 时机信号 | 15% | 买入信号6项 |
| 宏观匹配 | 10% | 环境7因素 |

**输出**: 前5-10名，使用 [report-templates.md](report-templates.md) 中的"选股筛选报告"模板。

## Mode B: 买卖时机 (Buy/Sell Timing)

### Step 1: 质量门控确认

**前提**: 标的必须通过Layer 2质量门控(财务≥5/7，护城河≥5分)

若未通过→告知用户: "时机分析对劣质股无意义，请先确认标的基本面质量。"

### Step 2: 深度技术分析

执行WebSearch获取技术数据：
```
- "[股票名] 技术分析 均线 MACD RSI {今日日期}"
- "[股票名] 成交量 量价分析 {今日日期}"
- "[股票名] 支撑位 压力位 关键点 {今日日期}"
- "[股票名] 近期新闻 催化剂 利好利空"
```

**A股特色技术数据** (必做):
```
- "[股票名] 龙虎榜 机构席位 {今日日期}"
- "[股票名] 换手率 量比 筹码 {今日日期}"
- "[股票名] 解禁 减持 {当前月份}"
- "[股票名] 连板 涨停 封板 {今日日期}"
```

**分析维度**:

| 指标 | 看多信号 | 看空信号 |
|------|---------|---------|
| 均线 | MA5>MA10>MA20>MA60 多头排列 | MA5<MA10<MA20<MA60 空头排列 |
| MACD | 零轴上方金叉，柱状图放大 | 零轴下方死叉，柱状图放大 |
| RSI | 30以下超卖区回升 | 70以上超买区回落 |
| 成交量 | 放量突破(>1.5倍均量) | 缩量上涨(量价背离) |
| 新闻催化 | 业绩超预期、政策利好 | 业绩不及预期、监管风险 |
| **龙虎榜** | 机构净买入，机构+游资同买 | 机构净卖出，游资独买 |
| **换手率** | 3-15%(活跃但不过度) | >25%(过度疯狂)或<1%(无人关注) |
| **量比** | 2.5-5(明显放量) | >10(极端放量见顶) |
| **筹码** | 股东户数连续减少 | 股东户数连续增加 |

### Step 3: 入场/出场点判定

**5种入场方法**:

| 方法 | 条件 | 适用场景 |
|------|------|----------|
| 关键点突破 | 放量突破盘整区间上沿 | Livermore核心方法 |
| 均线支撑 | 回调至MA20/MA60企稳 | 趋势中的回调买入 |
| MACD金叉 | 零轴上方金叉确认 | 中期趋势确认 |
| RSI超卖回升 | RSI从30以下回升 | 短期反弹 |
| 放量反转 | 大幅下跌后放量长阳 | 底部反转 |

**6种出场信号** (任一触发即卖出):

| 信号 | 条件 | 紧急程度 |
|------|------|----------|
| 护城河侵蚀 | 竞争加剧、技术颠覆 | 🔴 立即卖出 |
| 派发形态 | 高位放量滞涨，量价背离 | 🔴 立即卖出 |
| MACD死叉 | 零轴下方死叉 | 🟡 减仓 |
| RSI超买 | RSI从70以上回落 | 🟡 减仓 |
| 止损触发 | 从买入价下跌7-8% | 🔴 立即卖出 |
| 更好机会 | 发现更高性价比标的 | 🟢 换仓 |

### Step 4: 生成时机报告

判定结果: **BUY** / **WAIT** / **SELL** / **AVOID**

使用 [report-templates.md](report-templates.md) 中的"买卖时机报告"模板。

## Mode C: 持仓诊断 (Portfolio Review)

### Step 1: 收集持仓信息

**优先从 user-data/user-portfolio.md 自动读取**，无需用户重复输入。

```
Step 1.1: 读取用户持仓
  使用 Read 工具读取 user-data/user-portfolio.md:
  - 获取当前持仓列表(股票/代码/持股数/成本价/买入日期/买入逻辑/止损位/目标价)
  - 获取已清仓记录(用于归因分析)
  - 获取未卖出股票监控列表
  - 获取待补充信息

Step 1.2: 补充用户画像
  使用 Read 工具读取 user-data/user-profile.md:
  - 获取用户风险偏好
  - 获取行为特征(仓位管理/止损纪律等)
  - 获取持仓风格分析
  - 获取个性化建议

Step 1.3: 补充操作历史
  使用 Read 工具读取 user-data/trade-history.md:
  - 获取操作行为分析(止损执行率等)
  - 获取待确认操作
  - 获取分析建议执行跟踪

Step 1.4: 用户确认(可选)
  若 user-data/user-portfolio.md 中有"待补充"字段，则询问用户:
  - 买入日期
  - 买入逻辑
  - 是否有新增持仓未记录
  若用户提供了新的持仓信息，更新 user-data/user-portfolio.md

Step 1.5: 实时数据获取
  使用 WebSearch 获取每只持仓的当前价格和最新数据
```

若用户提供了 user-data/user-portfolio.md 中未记录的持仓，需更新持仓文件。

### Step 2: 逐只5层分析

对每只持仓执行完整5层分析，参考 [master-methods.md](master-methods.md)。

### Step 3: 组合层面评估

| 检查项 | 标准 | 未达标处理 |
|--------|------|-----------|
| 集中度 | 单股≤20%，单板块≤30% | 减仓超标持仓 |
| 现金储备 | ≥10% | 卖出最弱持仓 |
| 质量漂移 | 任一持仓未通过Layer 2 | 立即卖出 |
| 趋势恶化 | 任一持仓进入派发期 | 减仓或清仓 |
| 相关性 | 过度集中于同一宏观风险 | 分散化 |
| 林奇6月检查 | 增长逻辑是否仍成立 | 不成立则卖出 |

**(反馈闭环)**: 若质量漂移严重→优先处理最差持仓，而非均匀减仓。

### Step 4: 生成持仓报告

使用 [report-templates.md](report-templates.md) 中的"持仓诊断报告"模板。
