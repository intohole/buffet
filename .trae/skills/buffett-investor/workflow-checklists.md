# Workflow Checklists - 工作流检查清单

每个分析模式的工作流检查清单，确保步骤完整、不遗漏。

## 热点→选股漏斗 工作流

详细步骤参考: [hot-topic-mining.md](hot-topic-mining.md)

```
Phase 1: 热点信息采集
  ☐ 执行4批次并行WebSearch(市场面/政策面/业绩面/事件面)
  ☐ 执行A股特色数据搜索(连板梯队/龙虎榜/涨停分析/换手率排名)
  ☐ 识别3-5个候选热点
  ☐ 对每个候选热点执行5项纵深搜索
  ☐ (反馈闭环) 若纵深搜索发现业绩无法验证→降低评分或淘汰

Phase 2: 热点质量评估
  ☐ 对每个热点执行4维12分制评分
  ☐ 标记评级: ≥10分可持续 / 6-9分次级 / <6分回避
  ☐ (反馈闭环) 若所有热点<6分→告知用户当前无合适热点

Phase 3: 垂直领域确认
  ☐ 产业链定位(上/中/下游)
  ☐ 经济周期匹配
  ☐ 政策周期匹配
  ☐ 4项筛选标准检查(中军股/均线/资金/政策)
  ☐ 确定确定性等级(高/中/低)→对应仓位

Phase 4: 上市公司筛选
  ☐ 龙头/中军/补涨分类
  ☐ A股龙头战法增强检查(龙虎榜/连板质量/筹码/量比/题材收入占比)
  ☐ 5层框架快速筛选(每只标的)
  ☐ 割裂度筛选(板块背离/大盘背离/量价背离)
  ☐ (反馈闭环) 若无标的通过筛选→扩大搜索范围或降低标准

Phase 5: 股票深度分析
  ☐ 对重点标的执行7角度纵深搜索
  ☐ 完整5层深度分析
  ☐ A股红旗检查(14项必查，含退市/解禁/题材收入/龙虎榜)
  ☐ A股特色指标检查(龙虎榜/换手率/量比/筹码/连板/解禁)
  ☐ 计算内在价值和安全边际

Phase 6: 汇总输出
  ☐ 使用报告模板输出
  ☐ 包含评级+价格区间+止损位
  ☐ 包含风险提示
  ☐ 包含监控清单
```

## 选股筛选 工作流

详细步骤参考: [deep-analysis.md](deep-analysis.md) Mode A

```
Step 1: 确定筛选条件
  ☐ 风险偏好(保守/均衡/进取)
  ☐ 投资期限(短期/中期/长期)
  ☐ 板块偏好
  ☐ 资金规模

Step 2: 多源数据采集
  ☐ 执行4批次并行WebSearch(价值/成长/股息/板块)
  ☐ 收集10-20只候选标的

Step 3: 5层过滤
  ☐ Layer 1 风险与纪律(能力圈+情绪检查)
  ☐ Layer 2 门控(财务≥5/7，护城河≥5分)
  ☐ Layer 3 分类(林奇6型+PEG)
  ☐ Layer 4 时机(买入信号≥4/6)
  ☐ Layer 5 宏观(环境有利)
  ☐ (反馈闭环) 若Layer 2淘汰率>80%→放宽标准重新筛选

Step 4: 排名与选择
  ☐ 综合评分(护城河30%+财务25%+安全边际20%+时机15%+宏观10%)
  ☐ 输出前5-10名
  ☐ 生成选股筛选报告
```

## 买卖时机 工作流

详细步骤参考: [deep-analysis.md](deep-analysis.md) Mode B

```
Step 1: 质量门控确认
  ☐ 标的通过Layer 2质量门控
  ☐ 若未通过→告知用户"时机分析对劣质股无意义"

Step 2: 深度技术分析
  ☐ 获取当前股价及走势形态
  ☐ 关键均线(5/10/20/60/120MA)
  ☐ MACD状态
  ☐ RSI水平
  ☐ 成交量趋势
  ☐ 近期新闻催化
  ☐ A股特色数据(龙虎榜/换手率/量比/筹码/解禁)

Step 3: 入场/出场点判定
  ☐ 识别入场信号(关键点突破/均线支撑/MACD金叉/RSI超卖/放量反转)
  ☐ 识别出场信号(护城河侵蚀/派发形态/MACD死叉/RSI超买/止损触发)
  ☐ (反馈闭环) 若买入信号<4/6→判定WAIT，等待更多确认

Step 4: 生成时机报告
  ☐ 判定: BUY/WAIT/SELL/AVOID
  ☐ 入场区间+仓位+止损+目标价
  ☐ 风险提示
```

## 持仓诊断 工作流

详细步骤参考: [deep-analysis.md](deep-analysis.md) Mode C

```
Step 1: 收集持仓信息
  ☐ 读取 user-data/user-portfolio.md (持仓列表/统计/监控/待补充)
  ☐ 读取 user-data/user-profile.md (风险偏好/行为特征/持仓风格)
  ☐ 读取 user-data/trade-history.md (操作行为/待确认操作)
  ☐ 股票名称和代码
  ☐ 买入价格和日期
  ☐ 当前仓位占比
  ☐ 买入时的投资逻辑
  ☐ 提醒用户补充待补充信息

Step 2: 逐只5层分析
  ☐ 对每只持仓执行完整5层分析
  ☐ A股红旗检查(10项必查)

Step 3: 组合层面评估
  ☐ 集中度检查(单股≤20%，单板块≤30%)
  ☐ 现金储备(≥10%)
  ☐ 质量漂移(任一持仓未通过Layer 2→立即卖出)
  ☐ 趋势恶化(任一持仓进入派发期→减仓或清仓)
  ☐ 相关性(过度集中于同一宏观风险→分散化)
  ☐ (反馈闭环) 若质量漂移严重→优先处理最差持仓

Step 4: 生成持仓报告
  ☐ 每只持仓评级(卖出/减仓/持有/加仓)
  ☐ 调仓计划
  ☐ 风险提示
```

## 市场洞察 工作流

详细步骤参考: [sector-rotation.md](sector-rotation.md)

```
Step 1: 宏观数据采集
  ☐ 执行4批次并行WebSearch(市场面/政策面/全球面/估值面)

Step 2: 市场环境评分
  ☐ 7因素评分(利率/信用/情绪/政策/估值/成交量/趋势)
  ☐ 总分判定: ≥4多头 / 1-3中性 / ≤0空头

Step 3: 板块轮动分析
  ☐ 经济周期定位
  ☐ 板块强度排名
  ☐ 超配/标配/低配建议

Step 4: 反射性检查
  ☐ 泡沫检测清单(7项)
  ☐ 繁荣-萧条周期定位
  ☐ (反馈闭环) 若泡沫预警≥4项→建议全面减仓

Step 5: 哑铃策略配置
  ☐ 根据环境评分确定成长端/防御端/现金比例
  ☐ 具体板块配置建议

Step 6: 生成市场洞察报告
  ☐ 使用市场洞察报告模板
  ☐ 风险提示
```

## 复盘迭代 工作流

详细步骤参考: [review-loop.md](review-loop.md)

```
Step 1: 执行复盘(每次分析后)
  ☐ 检查数据完整性(必需数据点是否全部获取)
  ☐ 检查流程完整性(工作流检查清单是否全部完成)
  ☐ 检查判断一致性(5层框架的判断是否自洽)
  ☐ 检查输出规范性(报告是否按模板输出)

Step 2: 生成复盘文档
  ☐ 文件名: docs/review-YYYYMMDD-HHmm.md
  ☐ 填写分析概要(模式/标的/评级/入场价/止损/目标)
  ☐ 填写核心判断依据(5层框架的关键判断)
  ☐ 填写风险预警清单
  ☐ 填写待验证假设(哪些判断需要在后续验证)
  ☐ 填写经验沉淀(新发现/新技巧/避坑经验)

Step 3: 归因复盘(3-5个交易日后)
  ☐ 判断准确性: 评级与实际走势是否一致？
  ☐ 时机精确度: 入场价是否接近最优？
  ☐ 风险识别: 是否有遗漏的风险？
  ☐ 归因分类: 能力 vs 运气 / 执行偏差 vs 认知偏差 vs 系统性风险

Step 4: 策略复盘(每月1次)
  ☐ 胜率统计: 近期分析的胜率
  ☐ 方法论有效性: 哪层框架最准/最不准
  ☐ 策略迭代: 需要调整哪些阈值或规则
  ☐ 知识更新: 是否有新规律需要纳入
  ☐ 更新对应方法论文件(如有策略优化)
```
